iCoRe - The GDELT Interface for Communication Research

Getting Started

Welcome to iCoRe! iCoRe was designed to provide fast, intuitive, and scalable access to the Global Database of Events, Language and Tone (GDELT). The primary goal of iCoRe is the advancement of communication research centered around online news, event forecasting, and framing networks. This tutorial provides a short documentation of how to use and get started with iCoRe. For a more detailed discussion of iCoRe's features, capabilities, and applications, we recommend users to read the respective companion paper.
iCoRe is under active development. A list of soon upcoming features can be obtained here.

Technical Backend

iCoRe is based on a distributed database (Apache Cassandra) across a virtual cluster provided by the Aristotle Cloud Federation. iCoRe contains multiple tables to store the GDELT data according to different partition keys to speed up queries that span, for example, certain locations or time periods. Queries are driven by the DataStax Cassandra connector, with a lightweight front end written in Java and hosted in Apache Tomcat. Cassandra queries are enhanced by Apache Spark support, which allows for efficient complex queries, such as as filters and joins. Each query consists of an HTTP request, where the endpoint specifies the data source and the URL query string specifies the filters or constraints on the data. Users will be able to acquire data in CSV or JSON formats depending on need.
