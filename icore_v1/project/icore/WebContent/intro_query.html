iCoRe - The GDELT Interface for Communication Research

Query and Data Interpretation

iCoRe currently allows users to obtain data from GDELT's Global Knowledge Graph (GKG) and GDELT's Event (EVENTS) table. Compared to GDELT's own APIs, iCoRe does not limit the amount of data entries that can be pulled. Furthermore, iCoRe allows users to select specific content metrics instead of returning the complete set of content metrics that are contained in GDELT's Global Content Analysis Measurement (GCAM) string. Plus, iCoRe returns the complex structure of the GCAM string in a preprocessed, simple-to-use format. These features make iCoRe more efficient, user-friendly, and provide clearer insights into the data that are being returned.

After executing a query through iCoRe's interface, the data is returned in either CSV or JSON format. Below is a typical GKG query result in CSV format that is returned from iCoRe:

  • Themes: List of identified themes and topics that are present in a news article as identified by GDELT's Theme Dictionaries
  • Named Entities: List of identified named entities (e.g., persons, countries, organizations) that are present in a news article
  • Source: News source of the article. For a list of currently included sources in iCoRe, see
  • Wordcount: Article length of the news document
  • Source Location: FIPS country code of the news source location.
